## Code Review

### ❌ Critical

1. **`.githooks/pre-push` — `git add -A` in auto-fix path**
`git add -A` stages everything — `.env`, build artifacts, secrets. Use targeted staging:
`git add -- $(git diff --name-only)`

2. **`.githooks/pre-push` — wrong Claude CLI invocation for auto-fix**
Review uses `claude -p "$PROMPT"` but fix uses `claude --allowedTools "Edit" "$FIX_PROMPT"` (positional arg, no `-p`). Almost certainly broken. Should be `claude -p "$FIX_PROMPT" --allowedTools Edit`.

3. **`.githooks/pre-push` — `trap` overwritten inside loop**
`trap 'rm -f "$REVIEW_TEMP"' EXIT` is re-declared each iteration, referencing only the *latest* `$REVIEW_TEMP`. On a multi-ref push, all earlier temp files leak. Move the trap before the loop using an array, or delete `$REVIEW_TEMP` explicitly at the end of each iteration.

4. **`src/pages/ReviewDashboard.tsx:10-13` — `eager: true` bundles all `.txt` files**
Every review log is statically bundled. Bundle size grows unboundedly as logs accumulate — this silently bloats production. Use lazy loading:
```ts
const detailFiles = import.meta.glob('../../review-logs/details/*.txt', { query: '?raw', import: 'default' });
const content = await detailFiles[key]();
```

5. **`review-logs/reviews.csv` and `review-logs/details/*.txt` — PII in source control**
Real names, emails, and machine hostnames (`CludSmartzs-MacBook-Air.local`) are committed. Add `review-logs/` to `.gitignore` and scrub from history with `git filter-repo` or BFG.

### ⚠️ Improvements

6. **`.githooks/pre-push` — colons in `DETAIL_NAME` filename**
`${TIMESTAMP// /_}` replaces spaces but leaves colons: `2026-04-14_19:37:38_...txt`. Colons are invalid on Windows/FAT. Fix: `${TIMESTAMP//[: ]/_}`.

7. **`src/pages/ReviewDashboard.tsx:308-313` — stats not memoized**
`total`, `pushed`, `aborted`, `failed`, `notInstalled`, `uniqueDevs` recalculate on every render. Wrap in a single `useMemo(() => ..., [entries])`.

8. **`src/pages/ReviewDashboard.tsx:316-323` — `devBadge` defined inside component**
Recreated on every render. Move it outside `ReviewDashboard` as a pure function, same as `StatusBadge`.

9. **`src/pages/ReviewDashboard.tsx` — `expandedRow` not reset on filter change**
If a row is expanded, filtered out, then the filter is cleared — it reappears already expanded. Add `setExpandedRow(null)` to both `onChange` handlers for `search` and `statusFilter`.

10. **`src/pages/ReviewDashboard.tsx` — `window.location.reload()` is a no-op**
CSV and detail files are bundled at build time. A page reload doesn't pick up new data. Either remove the button or add a tooltip clarifying it's build-time data.

11. **`src/pages/Dashboard.tsx:4-7` and `src/pages/ReviewDashboard.tsx:17-50` — file-local types**
`User` (Dashboard.tsx), `ReviewEntry`, `ParsedIssue`, `DevStat`, `ChipColor` (ReviewDashboard.tsx) are all isolated. Move to `src/types/index.ts` for reuse and discoverability.

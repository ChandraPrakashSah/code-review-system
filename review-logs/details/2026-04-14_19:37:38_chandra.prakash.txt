## Code Review

### ❌ Critical

1. **`src/pages/ReviewDashboard.tsx` — `import.meta.glob` with `eager: true`**
   All `.txt` detail files are bundled at build time. As review logs accumulate, the JS bundle grows unboundedly — this will silently bloat production builds. Use lazy/dynamic loading instead:
   ```ts
   const detailFiles = import.meta.glob('../../review-logs/details/*.txt', { query: '?raw', import: 'default' });
   // then: const mod = await detailFiles[key]();
   ```

2. **`.githooks/pre-push` — `trap` overridden inside loop**
   `trap 'rm -f "$REVIEW_TEMP"' EXIT` is set inside the `for ref_line` loop. Each iteration overwrites the trap, referencing only the latest `$REVIEW_TEMP`. Temp files from previous iterations are leaked if the script exits mid-loop (multi-ref push). Set the trap once before the loop with an array or move cleanup inside the loop body.

   Despite `StatusBadge`/`StatCard`/`ReviewDetail` being extracted, the main `ReviewDashboard` JSX still embeds three major sections inline. Extract: `<StatsBar />`, `<DevComplianceTable />`, `<ActivityLog />`.
